# Teleprompter Voice Polish Design

**Date:** 2026-03-07
**Status:** Approved
**Goal:** Refine the teleprompter scripts so they sound more natural in a calm-founder speaking style while preserving the same proof structure and evidence discipline.

## Problem Statement

The teleprompter scripts are now structurally complete and evidence-aligned, but a few lines still sound slightly written, procedural, or too technical when spoken aloud. They work as recording aids, but they can be polished further so they feel more like a thoughtful solo builder speaking clearly on camera.

This polish pass is not about changing the content plan. It is about improving delivery.

## Chosen Direction

Optimize both teleprompters for a **calm founder** speaking style.

That means:

- steady cadence
- clear sentence rhythm
- low hype
- precise confidence
- natural transitions

It should sound like someone who built the system and understands it deeply, not like a product announcer or a generic startup pitch voice.

## Why This Direction Fits

The project is strongest when it feels intentional and credible.

The calm-founder style supports that by:

- keeping the technical proof believable
- making the demo easier to deliver without rushing
- helping the cloud-proof clip sound factual rather than procedural
- avoiding the “AI-generated marketing copy” feeling

## Scope

This polish pass is intentionally narrow.

### Files in scope

- `submission/demo-teleprompter.md`
- `submission/cloud-proof-teleprompter.md`

### Files out of scope

- `submission/demo-script.md`
- `submission/cloud-proof-script.md`
- `submission/README.md`
- screenshots, evidence files, and package structure

## Voice Rules

### Target tone

- thoughtful
- composed
- human
- founder-led
- technically grounded

### Preferred phrasing patterns

Use more lines like:

- “What I wanted here was...”
- “The important part is...”
- “I kept this step because...”
- “This is the proof moment for me...”

These sound natural while still carrying conviction.

### Phrasing to reduce

Reduce lines that sound like:

- rehearsed pitch copy
- credential recitation
- technical checklist narration
- long noun-heavy sentences

Examples of phrasing to soften:

- “The stack is intentionally Google end to end”
- “This short pass ties together...”
- any sentence that packs too many product names into one breath

## What to Improve

### 1. Sentence rhythm

Shorten or split sentences that are too dense for one take.

Focus on:

- breathability
- easier emphasis
- smoother on-camera delivery

### 2. Transition quality

Make transitions sound like spoken explanation rather than document structure.

For example:

- instead of abrupt section-to-section jumps, use small connective phrases
- move from setup to proof in a more conversational way

### 3. Technical phrasing

Keep the same proof points, but say them in a calmer, less list-like way.

The scripts should still mention:

- Gemini
- Google ADK
- Google GenAI SDK
- Cloud Run
- Google Cloud Text-to-Speech

But they should do so in spoken-friendly sentences, not as stacked label lists.

### 4. Closing strength

The ending lines should feel clean and confident.

They should not sound vague, overly dramatic, or like placeholder closing copy.

## Evidence Discipline

No new claims should be introduced in this polish pass.

The scripts must remain aligned with:

- the existing teleprompter structure
- the cue-sheet scripts
- the screenshots and Cloud Run proof bundle
- the current submission package language

This is a delivery polish pass, not a proof rewrite.

## Acceptance Criteria

The voice-polish pass is successful when:

- both teleprompter files sound more natural when read aloud
- the demo script feels easier to deliver in one take
- the cloud-proof script sounds calm and factual instead of procedural
- the scripts still match the current package evidence exactly
- the revised lines sound less AI-generic and more like the builder speaking directly
